Docker and Dockerhub

Containers

A container is a standard unit of software that packages up code and all its dependencies so the application runs quickly and reliably from one computing environment to another. A Docker container image is a lightweight, standalone, executable package of software that includes everything needed to run an application: code, runtime, system tools, system libraries and settings.

Images

Are the instruction set used to create containers. Once created they are immutable, meaning they cannot be changed and are used as a set of instructions to build apps. The same image can and should be reused in different projects for many reasons. One of them being that it saves the developer plenty of time by not having to rewrite monotonous code. It also offers a stable environment to test new code in. If the developer knows that the base image is well made, then they do not have to troubleshoot it if the problem arises. If the project is large enough, it can also save plenty of disk space as it will not be repeated throughout the project. An image needs to contain all of the frameworks and dependencies needed to run the application.

Docker Commands

  • docker –version
    • displays the current version of docker installed
  • docker pull
    • displays the current version of docker installed
  • docker run
    • used to create a container from an image
  • docker ps
    • used to display all running containers
  • docker ps -a
    • all containers active and those exited
  • docker exec -it bash
    • access running container
  • docker stop
    • stops running the container
  • docker kill
    • kills process
  • docker commit
    • creates a new image of an edited container
  • docker login
    • login to docker repository
  • docker login
    • login to docker repository
  • docker images
    • lists all locally stored images
  • docker rm
    • delete a stopped container
  • docker build
    • used to create an image from a specified folder

How to Start a project via command line

  1. Open terminal of choice
  2. Use command “cd ” to choose in which directory the folder should be located in. If no project exists, use either command “docker pull ” or “git clone ” to download the file.
  3. Use the cd command to enter the directory and then type “docker build -t
  4. docker run -dp 3000:3000
  5. Use “docker -it sh” to access terminal and run commands inside of the container

How to open a container via command line

  1. In cmd, use command “docker ps -a”
  2. This will show all containers that we ran. Copy the container id of the image required.
  3. Use command bash
  4. Can use a simple command like “ls” or “uptime”
