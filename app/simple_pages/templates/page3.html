Testing with Python/Flask

Pytest is a simple tool that can be used for different types of testing on the code that is to be deployed. For example, a test can be set up to verify that the home page loads by asking if an element is present. Pytest is then further integrated into the workflow through github in the workflows folder. Github actions lets a developer use their tests to control which versions can be merged with the main branch of the project.

Creating Simple Pages

The web pages were created using a base.html that contained the navbar and the copyright information. This was placed in static/templates folder because it is often called upon and the position cannot move. The simple pages folder then had a templates folder created inside with a .html file for each web page needed. The previously made base was then extended and the required text was filled in individually on the page. A style.css was then created in the static folder to better accommodate the HTML.
